Understanding the passwd Command

Basic Usage of passwd

The passwd command is used to change a user's password in Linux systems. It allows users to set or modify their own passwords and, when executed by the root user, can change passwords for any user.

passwd [OPTIONS] [USERNAME]

Common Options for passwd

The passwd command supports several options:

  • -d, --delete - Delete the user's password, allowing access without a password.
  • -l, --lock - Lock the user's password, preventing login.
  • -u, --unlock - Unlock the user's password, allowing login again.
  • -e, --expire - Expire the user's password immediately, forcing a change at the next login.
  • - Display the status of the user's password.
  • -h, --help - Display help information and exit.

Examples of passwd Command

Change Own Password

passwd

This command prompts the user to enter their current password and then the new password to update it.

Change Another User's Password

sudo passwd username

This command allows the root user to change the password for the specified username.

Lock a User's Password

sudo passwd -l username

This command locks the password for username, preventing the user from logging in until the password is unlocked.

Unlock a User's Password

sudo passwd -u username

This command unlocks the password for username, allowing them to log in again.

Expire a User's Password

sudo passwd -e username

This command expires the password for username, requiring them to change it at the next login.

Understanding Password Management

When a password is changed using the passwd command:

  • The new password is stored in an encrypted format in the /etc/shadow file.
  • Account locking prevents the user from logging in, even if they know the correct password.
  • Expiring a password forces the user to create a new one at the next login, enhancing security.
